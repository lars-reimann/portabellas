# I/O Discrepancies

## Polars has, Portabellas doesn't

### Read formats

| Polars | Notes |
|---|---|
| `read_csv` / `scan_csv` | CSV with many options (separator, dtypes, null_values, encoding, etc.) |
| `read_parquet` / `scan_parquet` | Parquet with many options (columns, memory_map, hive_partitioning, etc.) |
| `read_ipc` / `scan_ipc` | IPC/Feather format |
| `read_json` / `scan_json` | JSON format |
| `read_ndjson` / `scan_ndjson` | Newline-delimited JSON |
| `read_avro` | Apache Avro |
| `read_database` / `read_database_uri` | Database read (SQL, partitioning) |
| `read_excel` | Excel files |
| `read_delta` / `scan_delta` | Delta Lake |
| `read_iceberg` / `scan_iceberg` | Apache Iceberg |

### Write formats

| Polars | Notes |
|---|---|
| `write_csv` | CSV with many options (compression, separator, date formats, etc.) |
| `write_parquet` | Parquet with many options (compression, statistics, row groups, partition_by, etc.) |
| `write_ipc` / `write_ipc_stream` | IPC/Feather |
| `write_json` / `write_ndjson` | JSON/NDJSON |
| `write_avro` | Apache Avro |
| `write_database` | Database write |
| `write_delta` | Delta Lake write |
| `write_iceberg` | Iceberg write |
| `write_excel` | Excel with formatting options |
| `write_clipboard` | System clipboard |

### Streaming writes (LazyFrame)

| Polars | Notes |
|---|---|
| `sink_csv` / `sink_parquet` / `sink_ipc` / `sink_ndjson` | Streaming writes without collecting |
| `sink_delta` / `sink_iceberg` | Streaming writes to Delta/Iceberg |

### I/O infrastructure

| Polars | Notes |
|---|---|
| `ScanCastOptions` | Casting during scan |
| `PartitionBy` | Partitioned write config |
| `FileProviderArgs` | Cloud storage file providers |
| `CredentialProvider` / `GcpCredentialProvider` / `AwsCredentialProvider` / `AzureCredentialProvider` / `GenericCredentialProvider` | Cloud credential providers |
| `storage_options` | Cloud storage options on read/write |
| `from_arrow` / `from_dataframe` / `from_pandas` / `from_numpy` / `from_dicts` / `from_records` / `from_raw` | Conversion from other formats |
| `to_arrow` / `to_pandas` / `to_numpy` | Conversion to other formats |
| Serialize/deserialize | DataFrame and LazyFrame serialization |

## Portabellas has, Polars doesn't

| Portabellas | Notes |
|---|---|
| `Table.read.csv_file(path, *, separator)` | Simplified CSV read |
| `Table.read.json_file(path)` | Simplified JSON read |
| `Table.read.jsonl_file(path)` | Simplified JSONL read |
| `Table.read.parquet_file(path)` | Simplified Parquet read |
| `table.write.csv_file(path)` | Simplified CSV write |
| `table.write.json_file(path)` | Simplified JSON write |
| `table.write.jsonl_file(path)` | Simplified JSONL write |
| `table.write.parquet_file(path)` | Simplified Parquet write |

---

## Both have but with different APIs

| Feature | Portabellas | Polars | Difference |
|---|---|---|---|
| CSV read | `Table.read.csv_file(path, *, separator)` | `pl.read_csv(source, *, has_header, columns, separator, dtypes, null_values, ...)` | Polars has many more options |
| JSON read | `Table.read.json_file(path)` | `pl.read_json(source, *, infer_schema_length, schema, schema_overrides)` | Polars has schema options |
| NDJSON read | `Table.read.jsonl_file(path)` | `pl.read_ndjson(source, *, columns, infer_schema_length, schema, ...)` | Polars has more options |
| Parquet read | `Table.read.parquet_file(path)` | `pl.read_parquet(source, *, columns, n_rows, ...)` | Polars has many more options |
| CSV write | `table.write.csv_file(path)` | `df.write_csv(file, *, include_bom, compression, separator, date_format, ...)` | Polars has many more options |
| JSON write | `table.write.json_file(path)` | `df.write_json(file)` | Similar |
| NDJSON write | `table.write.jsonl_file(path)` | `df.write_ndjson(file, *, compression)` | Polars has compression |
| Parquet write | `table.write.parquet_file(path)` | `df.write_parquet(file, *, compression, statistics, row_group_size, ...)` | Polars has many more options |
| I/O style | `Table.read.*` / `table.write.*` | `pl.read_*` / `pl.scan_*` / `df.write_*` / `lf.sink_*` | Portabellas: methods on Table; Polars: module-level functions + instance methods |
